# Security Policy

## Supported Versions

The latest state of `main` is supported. Once `v0.1.0` is published, the latest
tagged release and `main` are the supported versions for security fixes.

## Reporting a Vulnerability

Use GitHub Security Advisories for private vulnerability reporting when
possible.

If private advisory reporting is unavailable, email
`fabian.barney@barney-media.com` with:

- the affected version or commit
- a short description of the issue
- reproduction steps or a proof of concept
- any suggested mitigations or impact notes

## Response Expectations

- Initial acknowledgement target: 3 business days
- Status update target after triage: 7 business days
